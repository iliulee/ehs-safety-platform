# 任务：彻底修复 AI 拆解 Header 编码错误（v5.0.2-4 复修）

## 背景
`defaultHeaders: { 'Content-Type': 'application/json' }` 已加但错误还在。
根因：OpenAI SDK 4.x 在 dangerouslyAllowBrowser 模式下，fetch 调用时
会塞 User-Agent 等 header 包含非 ISO-8859-1 字符（如中文项目名），
浏览器底层 Headers API 拒绝。

## 范围（只动 1 个文件）
`src/services/ai.service.ts`

## 修复方案（推荐 A）

### 方案 A：拦截 fetch 重写 header
在 createClient 内部，给 OpenAI 构造函数加一个 fetch 拦截器，强制把所有 header 转为 ASCII：

```typescript
private createClient(settings: AiSettings): OpenAI {
  return new OpenAI({
    baseURL: settings.baseUrl,
    apiKey: settings.apiKey,
    dangerouslyAllowBrowser: true,
    defaultHeaders: { 'Content-Type': 'application/json' },
    fetch: async (input, init) => {
      if (init?.headers) {
        const safeHeaders = new Headers()
        const headers = init.headers as Record<string, string>
        Object.entries(headers).forEach(([k, v]) => {
          const safeValue = String(v).replace(/[^\x00-\x7F]/g, '')
          safeHeaders.set(k, safeValue)
        })
        init.headers = safeHeaders
      }
      return fetch(input as any, init)
    },
  })
}
```
