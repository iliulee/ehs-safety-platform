# 新平台启动指南

## 使用步骤

把上面的提示词复制到新平台的对话中，让它先读这 5 个文件了解项目全貌，然后按照 V5.0 改造路线图继续开发。

我已经把这 5 个文件都保存好了，直接复制下面这段提示词到新平台粘贴即可：

---

```
【项目接续开发启动提示】

我是溜哥，我正在开发一个"安全管理平台"项目。这个项目之前用另一个 AI 编程工具开发到了 V5.0 改造阶段。现在我要切换到你这个平台继续开发。

项目代码在项目文件夹里。

请按以下步骤处理：

第一步：读取项目根目录下以下 5 个文件，全面了解项目
1. PROJECT-new_SPEC.md — 项目规格说明书（技术栈、架构、数据库、模块清单）
2. AGENTS.md — AI 代理角色定义（开发模式、代码规范）
3. VIBE_CODING_PROMPT.md — Vibe Coding 系统提示词（项目规则、开发约束）
4. 开发会话总结_V5.0改造.md — 历史会话总结（问题修复、待办事项）
5. 安全日志MVP开发提示词.md — 安全日志 MVP 的详细开发任务

第二步：阅读完以上文档后，确认你已经理解了：
- 项目的技术栈和架构
- 当前的待办事项和优先级
- 代码规范和质量要求

第三步：开始从 P0 优先级逐步推进开发：
Phase 1：修复变量生成关联项目 + 编辑器插入变量按钮
Phase 2：新增 workerSelect 变量类型 + WorkerSelectDialog 组件 + 安全日志 MVP
Phase 3：细节优化

请先读取这 5 个文件，然后告诉我你的理解，我们就可以开始了。
```
