# 任务：修复 AI 拆解字段缺失错误（v5.0.2-4 复修 3）

## 背景
AI 拆解能跑通了（Header 错误已修），但 zod 校验失败：
- hazards[0].date / title 必填但 AI 返回 undefined
- penalties[0].date / targetName 必填但 AI 返回 undefined

错误信息：
```
AI 返回格式异常：[ { code: invalid_type, expected: string,
received: undefined, path: [hazards, 0, date], message: Required },
{ code: invalid_type, expected: string, received: undefined,
path: [hazards, 0, title], message: Required },
{ code: invalid_type, expected: string, received: undefined,
path: [penalties, 0, date], message: Required },
{ code: invalid_type, expected: string, received: undefined,
path: [penalties, 0, targetName], message: Required } ]
```

## 根因
AI 把"沾边"的句子识别为隐患/处罚但没填完整字段，schema 直接报错。
两层防护：①prompt 严要求 ②解析时容错清洗。

## 范围（只动 1 个文件）
`src/services/ai-parser.service.ts`

## 修复方案（A + B 都做）

### 修复 1：parseDailyNarrative 里加容错清洗函数
在 `const parsed = JSON.parse(jsonStr)` 后、`Schema.parse(parsed)` 前调用：

```typescript
function cleanParsedData(data: any): any {
  // 过滤掉没填关键字段的隐患
  if (Array.isArray(data.hazards)) {
    data.hazards = data.hazards.filter(
      (h: any) => h && typeof h.date === 'string' && typeof h.title === 'string'
    )
    if (data.hazards.length === 0) delete data.hazards
  }
  // 过滤掉没填关键字段的处罚
  if (Array.isArray(data.penalties)) {
    data.penalties = data.penalties.filter(
      (p: any) => p && typeof p.date === 'string' && typeof p.targetName === 'string'
    )
    if (data.penalties.length === 0) delete data.penalties
  }
  return data
}
```

调用：
```typescript
const rawParsed = JSON.parse(jsonStr)
const cleaned = cleanParsedData(rawParsed)
return ParsedDailyDataSchema.parse(cleaned)
```

### 修复 2：buildPrompt 加严要求
在 buildPrompt 的"【要求】"段落末尾加：

```
- 隐患(hazards)：date 和 title 是必填，没填的整条不要写
- 处罚(penalties)：date 和 targetName 是必填，没填的整条不要写
- 不要写"{}"空对象或"未填写"占位符，没把握就整个数组不写
```
